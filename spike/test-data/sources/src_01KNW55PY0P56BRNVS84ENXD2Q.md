# 针对智能体的用户体验设计，第一部分：聊天 [译]

Reading notes and highlights from this source.

- 非流式聊天（响应是以完整的批次返回）的优缺点分析
  - 因为用户对非流式聊天可能没有立刻响应的预期，通常会允许Agent花费更多的时间来响应
  - 因为Agent在产生结果之前，内部会有很多步骤和过程，对于只获得结果的用户来说，是不透明的，需要用户给予一定的信任，才会派发任务给Agent
  - 在Agent完成上一个任务之前，很难接受用户发送新的消息（因为Agent可能已经规划并执行了部分过程，接受一条新的消息意味着中断和重新开始）
- 流式聊天（响应是以 token 流的方式返回）的优缺点分析
  - 流式的优点，用户可以实时地看到大模型输出内容，而不是卡住，降低了等待的焦虑
  - 聊天通常需要人类触发以及持续参与，无法独立完成需要较长时间运行的任务；现有的IM工具一般都没有支持流式聊天，接入会有一定的困难
  - 聊天交互的优点，因为大模型经常出错，用户可以聊天自然地纠正大模型错误理解，降低了使用门槛
