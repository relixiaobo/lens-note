# 科技爱好者周刊（第 341 期）：低代码编程，恐怕不会成功

Reading notes and highlights from this source.

- 作者认为低代码行不通的原因，在于先有UI，再有代码，代码是为了适配UI才生成出来的。
  - 但我的观点是，这并不能证明低代码行不通，因为在正常的产品设计和开发流程中，也是先有想法，再有原型，再有UI，最后才是代码，这个流程被证明是行得通的，而低代码之所以行不通，可能在于代码能力不够强？
