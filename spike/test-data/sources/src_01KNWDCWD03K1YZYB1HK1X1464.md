# Vibe-Coding as a VC: We Need to Eat Our Own Dog Food

Reading notes and highlights from this source.

- 没人愿意花时间去维护一个需要严格分类和结构化数据的CRM系统
  - 录入数据不便：录数据的员工不仅要学会使用复杂的系统，还要思考每个字段应该填什么
  - 录入工作的价值感不高，更像是"文书工作"，而非"核心工作"
  - 录入最好的时机是数据产生的时候，但极可能打断工作心流
  - 投入与回报的错位：录数据的员工不能直接即时的回报
    - 录入数据需要花费员工（比如销售人员）当下宝时间和精力
    - 数据整理好的好处，往往体现在管理层的报告里、公司未来的决策中，或是对整个团队的长期价值上。录入数据的人很难立刻看到录入数据的回报。
