# 我认识的最差程序员

Reading notes and highlights from this source.

- 使用单一指标衡量个人在团队中的价值，可能本身的前提就是错误的。
  - 有一些不可见的价值，无法通过传统的个人指标衡量，比如蒂姆通过结对编程交付了"一支能够持续交付高质量软件的团队"，个人指标无法捕捉到这种系统性价值
  - 真正的价值在于业务成果：应该衡量团队为组织带来的业务成果，比如创造了多少价值、节省了多少成本，而不是个人完成了多少用户故事
  - 团队作为一个复杂的系统，个体贡献和系统整体的表现之间不存在简单的线性关系（所有的个人贡献价值并不一定与业务的整体表现成正比，可能个人贡献都是正的，但是业务整体表现却是亏损）
  - 案例：看似"最差"的程序员
    - 真实价值：
      - 对新手：苏格拉底式引导，促进真正学习
      - 通过结对编程帮助团队中的每个人
      - 提升整个团队的代码质量和交付速度
      - 对资深开发者：思想碰撞，共同创造更优方案
    - 表面现象：
      - 管理层基于指标想要将其开除
      - 蒂姆·麦金农在个人绩效指标（交付的用户故事数）上始终为零
      - 从传统衡量标准看是"最差程序员"
