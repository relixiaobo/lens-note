# Google I/O '25 Keynote

Reading notes and highlights from this source.

- Google Assistant 的 3P框架（Personal, Proactive, Powerful）
  - Powerful 强大：具备创造和执行复杂任务的能力
    - 具体能力
      - 代码编程： 编写软件程序
      - 图像生成和编辑： 生成和修改图像
      - 视频创作： 使用Veo模型创建视频内容
      - 多模态理解： 理解和处理文本、图像、视频等多种内容
      - 网页交互： 通过Project Mariner在网页上执行任务
  - Personal 个性：真正了解用户的个人化助手
    - 实现方式
      - 提供私密、透明且完全由用户控制的个性化体验
      - 在用户授权下，整合Google生态系统的相关信息
      - 连接用户的Google搜索历史、Gmail等应用数据
      - 理解用户过去的行为模式（如搜索过的食谱）
    - 核心理念
      - 学习用户的偏好、项目和个人世界
      - 用户始终掌控主导权
      - 通过"个人上下文"(Personal Context)实现个性化
  - Proactive 主动：从被动响应转向主动预判和帮助
    - 核心理念
      - 在用户开口之前就能识别并解决问题
      - 不只是"你问它答"，而是能预见需求并提前准备
    - 具体场景
      - 生活助手： 根据上下文主动提供相关建议和提醒
      - 学习场景： 看到日历上有物理考试，提前一周主动准备个性化复习材料和测验
      - 日程管理： 主动识别日程冲突并提供解决方案
