# 模型即产品：未来 AI 产品演化的新范式

Reading notes and highlights from this source.

- 使用 强化学习 (RL) 的方式，让模型在一个模拟的互联网环境中通过反复试错来学习最佳的搜索策略
  - 监督微调 (Supervised Fine-Tuning, SFT): 先用一些已知的“好的搜索示例”（比如，专家是如何一步步找到某个信息的记录）来对模型进行初步训练。相当于"预热"或者是"新手指南"，让模型不是从完全随机的策略开始学习，可以加速后续强化学习的收敛速度，提高效率。
  - 强化学习多步训练: 模型（智能体/Agent）在这个模拟器中执行搜索任务。比如，给模型一个问题，它需要通过在模拟器中输入查询、点击链接等方式找到答案。如果模型采取的路径（一系列查询和点击）能高效地找到正确答案，就给予奖励；如果路径冗余、错误或找不到答案，就给予惩罚或低奖励。通过这种方式，模型会逐渐学会更优的搜索策略。
    - 模型的搜索行为（输入查询、点击链接）是行动 (Action)。
    - 模型当前所处的页面、已获取的信息是状态 (State)。
    - 模型就是智能体 (Agent)。
    - 模型学习到的“如何搜索”的策略是策略 (Policy)。
    - 根据搜索结果的好坏得到的反馈是奖励或惩罚（reward or penalty）。
  - 创建训练环境 (Environment): 创建一个“网络模拟器"（一个受控的、简化的、但又能反映真实网络某些特征的虚拟环境），有网页、链接、搜索框等，允许模型可以在里面进行“搜索”。
