# Meet Arc II of Arc Browser | A browser that browses for you

Reading notes and highlights from this source.

- Arc：预测用户需求，省去中间浏览过程，提供个性化结果 (A Browser that Browses for You.)
  - Deep Research
  - 竞品调研
  - Arc创始人表示，互联网的三个基本组成: Browser, Search Engine, Web Pages
    - 根据 Arc 浏览器创始人的说法，从用户视角来看，互联网的三个基本组成: Browser, Search Engine, Web Pages，或者说这三个基本组成就可以构成整个互联网
    - 我的理解
      - Web Pages：指的是信息或者应用本身
      - Browser：指的是用户访问信息或应用的载体
      - Search Engine：指的是用户获得信息或应用的途径
    - 也许将其修改为以下的观点可能会更加合理
    - 从用户视角来看，互联网的三个基本组成: 载体 + 途径 + 内容或应用
      - 也许还需要添加一个 关键部分 "用户"，指的是用户与其他所有部分的互动。
      - AI最可能在互联网哪部分带来关键的影响？
        - 内容或应用：动态生成，智能过滤，任务自动化
        - 途径：个性化的预测，而不是主动搜索，用户可能会更多地依赖AI来过滤和管理信息，从而更有效地分配注意力
        - 用户本身的参与感与存在感，可能会是用户体验的关键，机器的过度参与，是的人对于事情的全貌没有自己的概念，会导致失重感和价值感缺失。就像是当前的协作工作，每个人完成工作的一小部分，但谁也不知道自己的这个小部分的价值所在，导致工作上感受不佳。
        - 载体：语音交互的主要载体，更像是人与人的交互
      - 载体（工具或设备）：指的是用户用来访问互联网内容的工具或设备，比如浏览器、移动应用、智能设备等。这些载体是用户与互联网互动的界面。
      - 途径（方式或渠道）：指用户获取内容和服务的各种方式和渠道，这些途径是用户如何找到和获得他们所需信息或应用的方式。
        - 主动途径：用户主动寻找信息，如使用搜索引擎、直接输入网址等。
        - 被动途径：信息被推送给用户，如社交媒体feed、推荐系统等。
      - 内容或应用：指的是互联网实际承载的内容和功能，包括网页、视频、音频、在线服务、互动工具等。这些是用户最终在互联网中获取和使用的东西，互联网提供的实际价值所在。
      - Sider 属于改变了哪一部分？途径 还是 载体？还是什么都没改变？
  - 正常的浏览过程 / 代理浏览
    - 阅读不同网页的内容 --> 代理阅读
    - 浏览搜索结果页面 --> 代理浏览
    - 对比和筛选信息 --> 代理对比和筛选
    - 打开浏览器
    - 最终找到所需的信息或完成特定任务 --> 整合输出结果
    - 输入搜索关键词 --> 预测需求
    - 点击多个链接 --> 代理点击
  - 什么样的需求适合以 "AI" 为主？什么样的任务适合以 "人" 为主？
  - 但是对于本身以"浏览"为趣的用户，这样的设计可能并不友好。
