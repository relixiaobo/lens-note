think tool 与推理模型（reasoning model）是两种结构上不同的「思考」机制，区别不在能力，而在时态：推理模型是任务前的，处理「已有信息」并规划整体策略；think tool 是任务中的，处理「执行过程中新涌入的外部信息」（如工具调用结果）并动态调整策略。这个区分暗示了一个重要原则：静态推理不能替代动态推理。推理模型再强，也无法在任务开始前预见到它在执行中会遇到什么工具输出——因为那些输出此时还不存在。因此，think tool 不是推理模型的弱化版，而是对一个结构性盲点的填补：顺序决策中每一步之后出现的、需要当场消化的新事实。
