大多数人「用完即走、下次陌生人」的 AI 使用模式，本质上是在用有状态工具做无状态交互——每次对话都从零开始，积累的上下文全部丢失。Notion Agent 的方案（文档即规则 + 数据库即记忆）是一种具体的「状态外置」策略：把应该持久化的内容从对话上下文中移出，存入外部可寻址的结构（文档/数据库），让每次对话不再从零开始。这与知识库里「long-running agent 本质上是项目交接问题而非记忆问题」的洞察高度一致：解决方案不是拉长上下文窗口，而是工程化交接——文档和数据库正是这个「交接文件」的具体形态。
