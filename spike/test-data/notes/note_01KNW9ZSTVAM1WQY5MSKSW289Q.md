The Proactive scenario (calendar → study materials a week early, schedule conflict detection) adds a new dimension to the trust architecture described in note_01KNW5AYYB2GYNW169BHZYKQYD. That note decomposed trust into three temporal phases: pre-execution, during-execution, post-execution. But Proactive AI introduces a prior phase: *pre-initiation* — the agent must decide whether a need even exists before anyone has asked. A wrong prediction at this phase isn't just a trust failure, it's an attention and noise problem: if the agent surfaces unwanted materials or false conflict alerts, it degrades the very proactivity it was trying to provide. The trust architecture for proactive agents needs a fourth phase: calibrated prediction quality.
