Notion Agent 的调度场景（每周自动搜集资讯、每早扫描任务、每周五生成周报）是「主动型 AI」的一个特殊子集：触发条件是时间/日历，而非用户意图或外部事件。这和知识库里「Proactive AI 消除了意图表达步骤」的框架有一个微妙的区别：Notion 的定时任务是「规则驱动」的主动性（用户预先定义了触发规则），而 Google Assistant 的主动性是「推断驱动」的（模型从上下文推断出用户未表达的需求）。前者的主动性是可预期、可审计的——用户知道 Agent 什么时候会做什么；后者的主动性是涌现的、不透明的。这是两种不同的「主动性」，信任结构截然不同。
