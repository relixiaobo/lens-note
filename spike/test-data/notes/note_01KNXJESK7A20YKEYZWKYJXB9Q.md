Comparing LaCy's token-level delegation to the existing note on code vs. LLM reliability (note_01KNW9P12KASST6VJDGPJXG7DV): both make the same move — identify which computations are appropriate for the probabilistic LLM vs. which require a deterministic/authoritative source. The code note says 'wherever a task has an objectively verifiable answer, code is the epistemically appropriate tool.' LaCy says 'wherever a token has a factual ground truth that exceeds the model's capacity, delegation is the epistemically appropriate act.' These are two expressions of the same principle: route tasks to the substrate best suited to their epistemic profile. This may be a general principle of AI system design: every token/task/step should be executed by the component with the correct epistemic properties for that token/task/step.
