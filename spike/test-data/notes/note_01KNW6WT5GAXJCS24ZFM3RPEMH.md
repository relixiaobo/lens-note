The question 'when should you use multi-agent — is it always necessary?' is precisely what the existing knowledge base has been circling without ever landing on. The knowledge base has strong notes on *how* multi-agent works (token spend, context isolation, workflow patterns), and strong notes on *how to do it wrong* (role-based decomposition), but no explicit threshold criterion for *when it's warranted at all*. The title alone registers as a genuine gap — not an answered question dressed up as new content.
