The coding agent's read-environment-first ritual (pwd, git log, claude-progress.txt, feature_list.json) before selecting a task is a structured cold-start protocol that forces every session to reconstruct situational awareness from external state rather than from context memory. The ritual's value is not just informational — it enforces a rule that the agent cannot begin working until it can describe the current project state. This pre-task grounding directly counters the one-shot tendency: an agent that must first account for what exists is less likely to charge ahead and attempt everything at once.
