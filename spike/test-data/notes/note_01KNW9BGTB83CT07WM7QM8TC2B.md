The article's proposed architecture — [Agent Loop] + [MCP Servers] + [Skills] — maps a clean division of labor: MCP handles connectivity (agent ↔ external world), Skills handle expertise (agent ↔ domain knowledge). This clarifies the existing KB note on MCP as a 'vertical protocol.' MCP's vertical axis is about reach — can the agent access the data source or service at all? Skills' axis is orthogonal — does the agent know *what to do* once it has access? A system could have full MCP connectivity but be helpless without domain skills; and could have rich skills but be blind without MCP connections. The architecture separates two previously conflated concerns: reach and expertise.
