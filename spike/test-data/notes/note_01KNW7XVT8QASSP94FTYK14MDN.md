Agent context engineering has two fundamentally different cost types: idle cost (paid every turn regardless) and activation cost (paid only when a capability fires). L1 metadata is idle cost; L2 and L3 are activation costs. This asymmetry produces a practical design heuristic: when crafting a skill, the critical investment is not in the richness of L2 content but in the precision of the L1 description — because L1 is the one cost that cannot be amortized. A bloated L2 is paid only when that skill activates; a vague or over-broad L1 pollutes every single turn and degrades routing decisions for all other skills.
