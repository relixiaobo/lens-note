The claim that 'no single memory scheme fits all applications' is trivially true but productively constraining: it forces the question of which memory *type* is actually load-bearing for a given use case. An agent handling one-off factual queries is primarily semantic-memory-dependent. An agent managing a long-term project is episodic-memory-dependent. An agent executing skill-based tasks (coding, writing) is procedural-memory-dependent. Mismatching memory investment to actual dependency type is likely the most common memory design error — building elaborate episodic stores for an agent whose quality bottleneck is actually semantic knowledge gaps.
