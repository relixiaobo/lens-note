GAAT reveals that the existing trust architecture for agents (pre-execution help-seeking, during-execution transparency, post-execution correction — note_01KNW5AYYB2GYNW169BHZYKQYD) is a *user-facing* trust model, but enterprise multi-agent systems require a *system-facing* trust model operating at the infrastructure layer. The user-facing model helps a human decide whether to delegate to an agent; the system-facing model enforces policy invariants regardless of what agents do. These are not competing solutions — they address different principals. But the KB's existing trust architecture has an implicit gap: it assumes a single user who can observe, correct, and intervene. In a multi-agent system with dozens of autonomous agents, there is no human watching every interaction — the governance layer must be the 'observer-corrector' that the user cannot be.
