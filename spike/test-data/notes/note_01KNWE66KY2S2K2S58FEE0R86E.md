「截断并告知」比「截断并沉默」好，这是一个关于工具返回语义完整性的原则。对于大数据集（如 10000 行的表格），最差的做法是静默截断后直接返回前 N 行——Agent 会以为它看到了全部数据。正确做法是：返回样本行 + 元数据（总行数、截断原因）+ 如何获取更多数据的操作提示。这本质上是把「局部数据」和「全局元认知」同时传递给 Agent，让它知道自己处于数据的哪个位置，以及怎么继续探索。这与上下文工程中的「写下记忆」原则（让 Agent 感知自身状态）是同一逻辑在工具层面的体现。
