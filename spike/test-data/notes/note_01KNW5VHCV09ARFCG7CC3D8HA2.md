This RL-trained search agent architecture suggests a third path beyond the dichotomy in note_01KNVKYSKR2XJJNGSWTB35HQEM, which asks whether agent workflow patterns will be made obsolete by better base models or remain as permanent coordination infrastructure. The RL-search approach shows neither — it's models trained end-to-end to perform multi-step tasks via reward signals. The search workflow (query, click, refine, answer) isn't designed by a human architect and isn't an emergent property of scale — it is *learned through environmental feedback*. If this generalizes, the agent workflow taxonomy (orchestrator-worker, prompt chaining, etc.) may be a transitional artifact of the pretrain-only era, soon to be replaced by task-specific RL-trained policies.
