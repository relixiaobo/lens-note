Collaborative UX — where humans and AI agents work simultaneously in the same document, seeing each other's changes in real time — introduces a fundamentally harder coordination problem than sequential human-AI interaction. It requires conflict resolution, concurrent edit merging, and live progress visibility. This mirrors the git merge problem: parallelism creates divergent state that must be reconciled. The UX challenge isn't just 'showing AI work' — it's managing the moment two parties edit the same artifact at the same time, which is a distributed systems problem wrapped in a product design question.
