There is an implicit tension in the knowledge base between two framings of multi-agent necessity that the title's question sharpens: if multi-agent 'works mainly by spending more tokens' (the compute view), then the threshold for using it is purely economic — use it when the problem warrants more inference compute and you can afford it. But if multi-agent's primary purpose is context isolation (the engineering view), then the threshold is structural — use it only when context contamination would genuinely degrade a single-agent solution. These two criteria have different implications: the compute criterion is additive (more agents = better up to cost limit), the isolation criterion is surgical (more agents only when isolation is the bottleneck). A practitioner who conflates them will over-deploy multi-agent architecture on tasks that a single capable agent with a large context window could handle — a costly mistake as context windows grow.
