The advice to split a long SKILL.md when content is mutually exclusive adds a new dimension to the token-minimization logic of the three-level loading architecture. The existing L1/L2/L3 framework (note_01KNW7XVT38DPG0HX1X9M6EYPW) is a *vertical* partitioning strategy: load progressively deeper into one skill's content. The split-on-mutual-exclusion advice is a *horizontal* partitioning strategy: if a skill has two branches that are never used together, make them two separate skills — so each activation only loads the relevant branch. Together, vertical and horizontal partitioning form a complete content-organization theory for skill libraries: go deeper on demand (L1→L2→L3), and go narrower when content branches (one skill → two skills). The full principle is: load only what this specific task invocation will actually attend to.
