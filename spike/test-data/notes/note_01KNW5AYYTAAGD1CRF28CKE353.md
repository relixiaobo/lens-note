Parallel task execution is the structural advantage of background agents that has no analog in chat. Chat is inherently sequential — one thread, one task at a time. Background agents can dispatch multiple tasks simultaneously. This is not merely a speed optimization; it's a different model of human-agent collaboration: instead of the user directing one agent, they become a dispatcher managing a portfolio of concurrent executions. This shifts the UX challenge from 'how do I communicate with this agent' to 'how do I manage many agents' — a fundamentally different cognitive load problem.
