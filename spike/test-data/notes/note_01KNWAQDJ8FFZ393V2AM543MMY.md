Agents using search tools have a systematic over-specification bias: they tend to build very detailed, specific queries that return few or no results — essentially trying to retrieve the answer rather than retrieve evidence. The remedy is a search strategy reversal: start with short, broad queries to map the available information landscape, then progressively narrow. This is the difference between 'searching for a fact' and 'searching for a domain.' The over-specification failure mode is likely a reflection of how LLMs are trained — to produce precise, complete answers — being incorrectly transferred to the query-generation task, which benefits from imprecision and iteration.
