SKILL.md being the only mandatory file is a minimum viable skill principle: a skill needs only a name, description, and core instruction — everything else is optional scaffolding added when complexity demands it. This has a direct consequence for adoption: a new skill costs nothing more than a markdown file with frontmatter. The optional directories (scripts/, references/, assets/) exist to solve specific problems that arise as complexity grows, not as requirements from the start. This is the same progressive elaboration pattern seen in good software architecture: start minimal, add structure when a specific pain point justifies it.
