Pseudocode in prompts is best understood as a disambiguation layer rather than a control mechanism. Natural language instructions are semantically underdetermined — the same sentence can license multiple execution paths. Pseudocode collapses that ambiguity by borrowing the syntax of programming languages, which have unambiguous branching and sequencing semantics. The 'precision' gain from pseudocode prompting is therefore not primarily about adding new information — it's about reducing the model's interpretive degrees of freedom. This is structurally similar to what note_01KNW9BGT36Y40YGCWTEDWE7CE describes as code being 'self-documenting': the value comes not from the content but from the formalism constraining interpretation.
