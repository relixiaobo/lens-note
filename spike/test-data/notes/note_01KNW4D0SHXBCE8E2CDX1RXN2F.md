Manus's rejection of role-based multi-agent decomposition (Manager/Designer/Coder/Reviewer) is a concrete instance of the broader principle that agent workflow patterns should match task structure, not human organizational structure. Human role specialization exists because of cognitive limits and communication costs — a single person can't hold all context simultaneously. LLMs don't have the same constraint. The right decomposition axis isn't 'who does what' but 'what contaminates what' — i.e., where is context isolation genuinely needed?
