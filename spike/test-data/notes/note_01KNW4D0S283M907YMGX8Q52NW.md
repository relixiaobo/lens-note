Progressive disclosure applies to tool definitions just as well as to UI design. Manus's approach — tell the model 'what exists,' let it query 'how to use it' on demand — is the same principle that made complex software UIs learnable: don't show everything at once. The analogy runs deep: a tool schema dumped wholesale into context is like opening a full settings dialog on first launch. It overwhelms. The insight is that LLMs can tolerate deferred resolution — 'mcp-cli weather --help' is a valid runtime call, not a design failure. This reframes context budget as a UI resource, not just a technical constraint.
