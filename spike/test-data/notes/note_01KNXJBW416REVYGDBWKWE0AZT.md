The conference program session title pairing is telling: Apple's UI generation paper is placed in a session called 'Personalization and Human-AI Alignment.' This framing — treating UI generation as an alignment problem — signals a shift in how the HCI research community is categorizing design automation. It's no longer 'tools that generate interfaces' (a capabilities frame) but 'AI systems that need to align with designer intent' (a values/preference frame). This is the same conceptual move that transformed ML from 'systems that optimize metrics' to 'systems that need to align with human values.' Applying alignment framing to UX design suggests the community expects AI design tools to develop the same misalignment failure modes as language models.
