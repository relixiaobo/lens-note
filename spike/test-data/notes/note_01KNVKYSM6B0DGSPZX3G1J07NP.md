The 'voting' variant of parallelization is an ensemble method applied to LLM judgment — multiple agents run the same task independently to gain confidence through redundancy. This is interesting from a Bitter Lesson perspective: ensemble/voting is a general statistical method (not domain knowledge), but the 'different perspectives' framing implies human-designed prompt diversity. Whether this is genuine generality or disguised knowledge-injection depends on how the perspectives are constructed. If they are task-agnostic, it's a general method; if curated for the domain, it's knowledge injection.
