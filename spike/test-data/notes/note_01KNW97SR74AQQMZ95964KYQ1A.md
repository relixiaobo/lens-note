Progressive disclosure and the layered tool architecture are two implementations of the same underlying insight applied to different parts of the tool-knowledge problem: *what tools exist* vs. *how each tool works*. Progressive disclosure (mcp-cli list → mcp-cli weather --help) addresses the 'how to use it' problem — capability discovery on demand. The layered architecture (L1 schema in context, L2 via bash, L3 via script) addresses the 'what is available' problem — capability exposure at different costs. Together they form a complete theory of tool context management: the *set* of tools is managed by layer, and the *schema depth* of each tool is managed by disclosure timing. Neither alone would be sufficient.
