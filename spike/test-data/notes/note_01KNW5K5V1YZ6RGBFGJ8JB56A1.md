低代码工具失败的真正瓶颈，很可能不是「开发流程方向」（先 UI 还是先代码），而是「代码生成能力的上限」。传统低代码工具用可视化组件生成的代码往往是有限领域的固定模板，无法处理真实产品的复杂逻辑。随着 LLM 代码能力提升，这个瓶颈正在被打破——这意味着低代码的历史性失败是一个「能力不足时代的判断」，不一定适用于 AI 编程时代。
