The evaluator-optimizer workflow is structurally identical to software's quality-improvement loop: a producer generates output, a critic identifies gaps, iteration converges on a standard. Fowler's claim that high-quality software emerges from cycles of feedback and refinement applies directly here. The key insight is that this pattern only works when two conditions co-occur: the LLM can improve given feedback (trainability), and another LLM can reliably generate that feedback (evaluability). When either condition fails, the loop stalls or diverges.
