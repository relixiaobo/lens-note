Streaming Chat's core UX value is anxiety reduction, not speed. The article notes users can see token output in real time 'without freezing' — but this doesn't make the task complete faster. What it does is reframe waiting: from 'is anything happening?' to 'I can see it working.' This is the same principle as progress bars in file transfers — they don't speed up the transfer, they manage perceived wait time. The implication for Agent UX design is that *visibility of process* is often more important than *speed of completion*.
