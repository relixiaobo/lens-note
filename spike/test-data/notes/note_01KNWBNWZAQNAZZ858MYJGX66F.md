The existing agent UX taxonomy (note_01KNW55PVWCE5BT22R10B57K1W) maps paradigms along two axes: attention required and locus of control. Arc II exposes a missing third axis: epistemic depth — does the user end up understanding the landscape that produced the answer, or only the answer itself? Ambient UX optimizes for attention, treating focus as the scarce resource. But for knowledge work, the scarcer resource is often *understanding*, not focus. These two can be inversely correlated: the more autonomously an agent retrieves and synthesizes, the less the user learns about the problem space. The right UX paradigm for a task depends on whether the journey or the destination is what the user actually needs — and this is orthogonal to how much attention the user has available.
