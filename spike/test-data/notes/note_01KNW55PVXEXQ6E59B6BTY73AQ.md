Generative UI — where AI directly emits HTML or maps its output to UI components — inverts the traditional software development flow. Normally, humans design the interface and AI fills in content. In generative UI, AI designs the interface and the content simultaneously. The 'component mapping' variant (AI selects from predefined components) is actually a constrained version that preserves human design authority, while 'full generation' (raw HTML) surrenders it entirely. This creates a new alignment-style question at the UX layer: do you trust AI to make presentation decisions, or only content decisions?
