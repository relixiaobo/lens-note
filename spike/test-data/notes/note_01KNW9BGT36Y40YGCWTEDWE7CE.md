The insight that code-as-skill is superior to natural-language instructions-as-tool because it's 'self-documenting and modifiable' extends the existing KB argument about engineered vs. prompt-based planning — but with a crucial additional property: agent-modifiability. The existing note argues that code is more stable than prompts because prompts get diluted as context grows. This article adds: code inside a skill can be *modified by the agent itself* when it's struggling. Natural language tool descriptions are opaque to the agent that uses them — the agent cannot change them. Code is transparent and editable. This means Skills create a feedback loop: the agent can improve the tool during use, not just invoke it. This is a fundamentally different relationship between agent and capability.
