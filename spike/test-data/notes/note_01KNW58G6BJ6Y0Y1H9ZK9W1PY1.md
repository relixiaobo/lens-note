Non-streaming Agent responses create an opacity problem that requires a trust prerequisite before users will delegate tasks. Because users cannot see intermediate steps, they must trust the agent blindly during execution. This is the same dynamic as delegating to a human subordinate you've never worked with: you won't give them a complex task until you've seen how they handle something smaller and transparent. Streaming partially dissolves this by making steps visible — it's the difference between a black-box service and a glass-box process.
